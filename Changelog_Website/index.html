DBFA Changelog


This page tracks the changelog for the latest build of DBFA.
 
Latest release: Inception 1 Patch 1
 

Presenting SUPERCACHING

- Presenting SUPERCACHING: Unlike other codes which refer db values for every product, thereby adding time delay, DBFA fetches all details when the billing is started, and caches the data in its memory, thereby greatly increasing speed and reliability. SUPERCACHING is dynamically refreshed whenever the source is changed.

Posted by Start Bootstrap on September 24, 2018

Changes to Billing

- Remove last-added product from cart: Type in "lastrem" to remove the last-added product from cart.

- Bill-splitting in DBFA: DBFA Billing now has an option to split-bill. Decimal values are rounded off to two digits.

-  Live-cart listing: DBFA now lists all in-cart products during product selection.

Revamped Store Reporter

- DBFA Security Status: DBFA now scans itself for security, integrity and web-hook health. The same is beautifully added to the store report. Voucher and Delivery Stats have also been added.

- Employee Attendance Metrics: The reporter now shows employee attendance metrics.

- Sales Records: The reporter now shows sales logs from the last one month (one month from report generation data) 
 
- delta Deep Archival Engine Data: The report now also shows last month's archival data (incomplete keys are shows for security purposes!)
